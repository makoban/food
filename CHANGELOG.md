# CHANGELOG - AI飲食店エリア分析

## v1.1 (2026-02-13)

### エリア名入力対応

- **エリア直接入力**: URLに加え、都道府県名・市区町村名でも分析可能に
  - 入力欄にエリア名（例: 渋谷区、名古屋市）を入力するとエリア飲食市場分析を実行
  - URLを入力した場合は従来通りWebサイト巡回+AI分析を実行
- **area-database.js追加**: 全国1,800+市区町村のデータベースを搭載（house_searchから移植）
- **エリア選択モーダル**: 同名エリアが複数ある場合、選択モーダルを表示
- **エリア専用分析フロー**: `startAreaOnlyAnalysis()` — Webサイト巡回をスキップし、e-Stat + Geminiで飲食市場データを即座に分析

## v1.0 (2026-02-13)

### 初回リリース

- searchAI v4.0（全業種対応版）をベースに飲食業界特化版として新規作成
- **業種固定**: 全業種対応の業種判定ロジックを削除し、飲食業（restaurant）に固定
- **UIテーマ**: 赤系カラー（#ef4444 → #f97316）の飲食テーマに変更
- **飲食業専用プロンプト**: Gemini向けプロンプトを飲食業に最適化
  - 外食支出額、飲食店密度、ランチ/ディナー需要、客単価、席回転率
  - フードデリバリー需要、テイクアウト比率
- **飲食業専用KPI**: 外食支出、飲食店密度、人口あたり店舗数、世帯消費傾向
- **e-Statデータセット**: `0003348239`（家計調査・外食）を使用
- **飲食業専用レポート**:
  - 飲食市場データ（月間外食支出・年間外食支出・食費比率）
  - 競合分析（飲食店数・人口1万人あたり店舗数・チェーン比率）
  - 消費者プロファイル（世帯収入・単身世帯率・オフィスワーカー密度）
  - 市場ポテンシャル（ターゲット人口・日次歩行者数・ランチ需要・ディナー需要）
- **Chart.jsグラフ**: 人口×外食支出、飲食店数×ランチ需要 の2軸グラフ
- **Excel出力**: 飲食業専用のシート構成（全店サマリー・店舗概要・巡回ページ・エリア別詳細）
- **APIプロキシ**: 既存の `house-search-proxy.ai-fudosan.workers.dev` を共有
- **外部JSファイル不要**: `industry-config.js` と `industry-prompts.js` を削除し、app.js に統合
